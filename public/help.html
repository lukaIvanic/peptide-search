Help & Quickstart

How to use the app efficiently

Dashboard Entity Explorer Baseline

Quickstart

  • Search on the dashboard and start extraction to create runs.
  • Open a run to see entities, prompts, and raw JSON.
  • Filter the papers table by status or source to focus QA.
  • Use Export CSV to snapshot the current filtered list.
  • Use Entity Explorer for QA across all runs.
  • Edit a run to create a new version (append-only).
  • Use “Start tour” in the header for guided walkthroughs.

Entity Explorer

  • Filter by provider, prompt version, source, and flags.
  • Use Latest run only and Recent (minutes) to focus new data.
  • Use Evidence Gaps view to see which fields lack evidence most often.
  • When grouping by SMILES/InChI, enable Show missing key to see (missing).
  • Review mode surfaces flagged entities and provides next/random stepping.
  • Keyboard shortcuts: N (next), R (random), E (toggle evidence gaps).
  • Prompt comparison shows KPI deltas between two prompt versions.

Run Details & Editor

  • Run page shows entities, prompts, and raw JSON.
  • Compare versions defaults to older → newer and labels show order.
  • Changes & Evidence uses field-level evidence for WHY.
  • Follow-up chat creates a new run version from prior context.
  • If streaming tokens are unavailable, a status message is shown.
  • Editor creates append-only versions with reasons and evidence updates.

Failure triage

  • Use the Failure breakdown panel on the dashboard to spot clusters.
  • Click View on any bucket to open a drilldown list of runs.
  • Open a sample failed run directly from each category or reason.
  • Use “Retry list” to re-queue all failures in the drilldown.
  • Adjust the window to 7/30/90 days to focus your diagnosis.

Keyboard shortcuts

  • / focus search input.
  • F focus papers filter input.
  • Shift + F filter to Failed.
  • Shift + P filter to Processing.
  • Shift + N filter to No runs.
  • Shift + C clear filters.
  • Shift + E export CSV (failure modal takes precedence).
  • Esc close the failure drilldown.

Quality Rules

Use the Quality Rules panel to tune validation thresholds (pH, temperature, invalid sequences, missing evidence) and observe how flags/KPIs change.

Troubleshooting

  • If changes are not visible, refresh and check cache-busting parameters.
  • If the server appears stuck, restart `uvicorn app.main:app` (no reload for stability).
  • Evidence display requires `entity_index` to be populated (newer runs).

Danger zone

This deletes all extracted runs, entities, and papers from the local database. Baseline files remain untouched.
